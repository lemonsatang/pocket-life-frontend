# 통계 페이지 API 404/405 에러 관련 문의

## 문제 상황
통계 페이지에서 다음 API 호출 시 404 또는 405 에러가 발생합니다:

1. `GET /api/stats/meal/range` - 404 Not Found
2. `GET /api/stats/cart/range` - 404 Not Found
3. `GET /api/todo/stats` - 404 Not Found
4. `GET /api/tx/category-stats` - 405 Method Not Allowed

## 요청 사항

### 1. API 엔드포인트 확인
이전에 제공받은 API 문서(front_md/2번.md)에 따르면 다음 API들이 구현되어 있다고 했습니다:
- `/api/stats/meal/range?startDate=...&endDate=...`
- `/api/stats/cart/range?startDate=...&endDate=...`
- `/api/todo/stats?startDate=...&endDate=...`
- `/api/tx/category-stats?startDate=...&endDate=...`

**현재 이 API들이 실제로 구현되어 있는지 확인이 필요합니다.**

### 2. 대안 API 확인
만약 위 API들이 아직 구현되지 않았다면, 다음 대안을 사용할 수 있는지 확인이 필요합니다:

#### 식단 통계
- 기존 `/api/meals?date=...` API로 기간 범위 조회가 가능한지?
- 또는 `/api/meals?startDate=...&endDate=...` 형식 지원 여부?

#### 장바구니 통계
- 기존 `/api/cart?date=...` API로 기간 범위 조회가 가능한지?
- 또는 `/api/cart?startDate=...&endDate=...` 형식 지원 여부?

#### 일정 통계
- 기존 `/api/todo/getList?date=...` API로 기간 범위 조회가 가능한지?
- 또는 `/api/todo/getList?startDate=...&endDate=...` 형식 지원 여부?

#### 카테고리별 지출 통계
- `/api/tx/category-stats`의 HTTP 메서드가 GET이 맞는지?
- 또는 다른 엔드포인트를 사용해야 하는지?

### 3. 데이터 반영 문제
식단관리 페이지와 장바구니 페이지에 작성된 내용이 통계 페이지에 반영되지 않는 문제가 있습니다.
- 통계 페이지에서 조회하는 데이터가 실제 저장된 데이터와 일치하는지 확인이 필요합니다.
- 날짜 필터링이 제대로 작동하는지 확인이 필요합니다.

## 참고
- 현재 프론트엔드는 기간 범위(startDate, endDate)로 데이터를 조회하려고 합니다.
- 주 단위: 월요일~일요일
- 월 단위: 해당 월의 1일~말일
