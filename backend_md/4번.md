# 통계 페이지 식단 데이터 미반영 문제 문의

## 문제 상황
식단관리 페이지에 1월 20일 데이터가 기록되어 있지만, 통계 페이지의 식단관리 차트에 데이터가 표시되지 않습니다.

## 확인 사항

### 1. API 엔드포인트 확인
통계 페이지에서 사용하는 API:
- `GET /api/stats/meal/range?startDate=2026-01-19&endDate=2026-01-25` (주 단위)
- `GET /api/stats/meal/range?startDate=2026-01-01&endDate=2026-01-31` (월 단위)

식단관리 페이지에서 사용하는 API:
- `GET /api/meals?date=2026-01-20`

**질문:**
1. `/api/stats/meal/range` API가 `/api/meals`에 저장된 데이터를 포함하여 집계하는지?
2. 식단 데이터의 날짜 필드명이 `mealDate`인지 확인이 필요합니다.

### 2. 날짜 필드명 확인
식단 데이터 저장 시 사용하는 날짜 필드명이 무엇인지 확인이 필요합니다.
- `mealDate`?
- `date`?
- 다른 필드명?

### 3. 날짜 범위 필터링 확인
`/api/stats/meal/range` API에서 날짜 범위 필터링이 올바르게 작동하는지 확인이 필요합니다.
- `startDate`와 `endDate` 사이의 데이터를 모두 포함하는지?
- 날짜 비교 시 시간 부분은 무시하고 날짜만 비교하는지?

### 4. 데이터 저장 확인
1월 20일에 저장된 식단 데이터가 실제로 DB에 저장되어 있는지 확인이 필요합니다.

## 테스트 요청
다음 API 호출 결과를 확인해주세요:
```
GET /api/stats/meal/range?startDate=2026-01-19&endDate=2026-01-25
GET /api/stats/meal?date=2026-01-20
GET /api/meals?date=2026-01-20
```

각 API의 응답 데이터를 확인하여 데이터가 올바르게 조회되는지 확인이 필요합니다.
